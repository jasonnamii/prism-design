# Layout Catalog — HTML 레이아웃 정본

> 레이아웃은 *모양*이 아니라 *읽히는 방식*이다. 카탈로그는 *어휘 사전*·정답 ✗.

## 핵심 원칙

→ **콘텐츠 역할 → 레이아웃 archetype 선택.** "예쁜 배치" ✗·"이 섹션이 어떻게 읽히는가" ○.
→ **한 페이지 = 다른 레이아웃 mix**: Hero → Explanation → Evidence → System → Decision → CTA
→ **같은 레이아웃 archetype 2회 이상 반복 ✗** — 카드 다음 카드 ✗·이미지-텍스트 좌우 반복 ✗

## 1. Hero / Opening Layouts

| Archetype | 정의·사용 | 피할 때 |
|---|---|---|
| **Full-bleed Hero** | 단일 이미지/영상/시각 필드가 viewport 채움 → 브랜드 오프닝·캠페인 랜딩·프리미엄·시네마틱 | 복잡 정보 즉시 전달 필요 |
| **Centered Hero** | 헤드라인·서브·CTA 중앙 → 단순 랜딩·명확 가치 제안·MVP. **★ AI 남용 주의 — 매우 generic** | 프리미엄 아트 디렉션 필요 |
| **Split Hero** | 텍스트 / 이미지·제품·증거 좌우 → 제품 설명·비즈 랜딩·앱 intro | 더 에디토리얼/감성 오프닝 필요 |
| **Asymmetric Hero** | 의도적 off-balance·강한 네거티브 스페이스 → 프리미엄 에디토리얼·founder letter·culture·고급 문서 | 단순 전환 페이지 (clarity 최우선) |
| **Product Stage Hero** | 제품/작품/사물이 *주인공*으로 무대화 → Apple식 제품·앱·갤러리 입구 | 시각 자산 약함 |
| **Manifesto Hero** | 강한 선언이 첫 화면 지배 → 운동·CEO 타운홀·브랜드 belief·문화 thesis | 디테일 설명 필요 |
| **Layered Hero** | 텍스트·subject·배경 레이어 겹침 → 프리미엄 비주얼 아이덴티티·creator·잠금화면식 | 텍스트 가독성 critical |

## 2. Editorial / Document Layouts

|---|---|---|
| **Magazine Spread** | 큰 타이틀·이미지·캡션·컬럼 → 문화·교육·long-form·founder narrative | 순수 대시보드·트랜잭션 UI |

| **Side Rail Layout** | 고정 사이드 컬럼·내비/메타/섹션 라벨 → 긴 HTML 문서·리포트·전략 메모·KB | 단순 1페이지 랜딩 |
| **Annotated Document** | 본문 + 사이드 노트·주석·footnote·증거 → 전문가 리포트·legal/business 리뷰·교육 철학 | 소비자 캠페인 |
| **Research Dossier** | 증거·인용·타임라인·라벨·요약 *연구 파일*처럼 → 심층 리서치·시장 분석·역사/문화 | 깨끗한 제품 마케팅 |
| **Academic Paper** | 타이틀·abstract·섹션·레퍼런스·표·다이어그램 → 진지한 thought leadership·교육 모델·방법론 | 감성 브랜드 스토리텔링 |
| **Executive Memo** | 결정 지향·문제·옵션·추천·근거·리스크 → CEO 결정·이사회·투자 결정·legal/biz 선택 | 브랜드 랜딩 |

## 3. Grid / Card Systems

| Archetype | 정의·사용 | 피할 때 |
|---|---|---|
| **Bento Grid** | 크기 다른 카드 모듈 정보 그리드 → feature 오버뷰·플랫폼 capabilities·제품 요약. **★ AI 남용 — 위계 없으면 generic** | 모든 페이지 (AI 디폴트) |
| **Uniform Card Grid** | 동일 크기 카드·병렬 항목 → feature·benefit·team·카테고리 | 프리미엄 차별화 필요 |
| **Masonry Grid** | 다른 높이 항목·컬럼 배치 → 포트폴리오·inspiration·이미지 갤러리 | 정밀 비교·동등 위계 |
| **Modular Grid** | 엄격 모듈러·정렬·반복 유닛 → 프로 리포트·디자인 시스템·기관 사이트 | 표현적 캠페인 |
| **Card Stack** | 물리 시트처럼 카드 겹침 → 로드맵·증거 stack·앱 화면·layered 옵션 | 다수 항목 스캔 필요 |
| **Tile Wall** | 밀집 그리드·이미지·영상·로고·예시 → ecosystem·creator·proof of activity | 단일 프리미엄 사물 포커스 필요 |
| **Comparison Cards** | 대안을 병렬 카드로 결정 → 2-5 전략·제품·패키지·시나리오 비교 | 많은 nuanced 변수 (matrix 사용) |

## 5. Dashboard / Data Layouts

| Archetype | 정의·사용 | 피할 때 |
|---|---|---|
| **KPI Grid** | 핵심 수치를 간결 그리드 → 시장 규모·성과·traction·운영 상태 | 수치 약함·근거 없음 |
| **Command Center** | 다수 라이브/상태 모듈·고수준 운영 뷰 → 플랫폼 ops·내부 관리·AI 모니터링 | 공개 스토리텔링. **★ fake dashboard 위험** |
| **Data Wall** | 차트·도형·맵·증거 블록·밀집 분석 → 심층 시장·트렌드·투자 thesis | 임원 요약 |

| **Scorecard Layout** | 옵션을 기준 점수화 → 벤더 비교·전략 옵션·후보 평가 | 정성 narrative 우선 |
| **Map Layout** | 카테고리·관계 공간 정리 → 경쟁 landscape·ecosystem·stakeholder | 선형 스토리 필요 |

## 7. Decision / Strategic Layouts

| Archetype | 정의·사용 | 피할 때 |
|---|---|---|
| **Pyramid Layout** | 결론 우선·근거·증거 → 임원 결정·투자 메모·전략 추천 | 탐색적 창작 ideation |
| **Option Matrix** | 옵션을 기준 매트릭스 비교 → 2-5 전략 경로 선택 | nuanced 변수 많음 |

| **Decision Box** | 핵심 결정·추천 시각적 isolation → CEO 메모·이사회·legal 추천 | 감성 브랜드 |

## 13. 조합 레시피

### A. Apple식 제품 랜딩
```
Product Stage Hero → Sticky Feature Story → Visual Stack →
Minimal Spec Grid → Sticky Compare Panel → Final CTA
```
**Avoid**: 3개 동일 feature 카드·neon gradient 배경·fake dashboard

### B. 고급 전략 리포트
```
Executive Memo → Pyramid → KPI Grid → Option Matrix →
Risk-Reward Map → Decision Box
```
**Avoid**: 시네마틱 parallax·포스터 타이포·과디자인 카드

### C. KISAS식 교육 철학
```
Manifesto Hero → Magazine Spread → Annotated Document →
Step Flow → Timeline → Research Dossier → Decision/Definition Box
```
**Avoid**: generic SaaS bento·dark futuristic AI 스타일

### D. Cre8orClub 갤러리
```
Spotlight Stage → Curtain Reveal → Museum Wall →
Progressive Disclosure → Contact Sheet → Object Catalog
```
**Avoid**: infinite feed·generic thumbnail grid·commerce 카드

### E. 문화/캠페인 랜딩
```
Full-bleed Hero → Poster Layout → Storyboard →
Collage → Social Proof → Landing Funnel CTA
```
**Avoid**: 기업 대시보드·과한 깨끗 SaaS 레이아웃

## 14. Anti-Generic Layout 룰

```
✗ centered hero + 3 feature cards
✗ identical card grid 모든 섹션
✗ 이미지-좌·텍스트-우 2회 이상 반복
✗ bento grid 위계 없이
✗ fake dashboard 스크린샷
✗ generic testimonial 카드
✗ pricing card (가격이 주 결정 ✗일 때)
✗ excessive rounded card
✗ 모든 섹션 동일 max-width container
✗ 모든 섹션 동일 vertical 리듬
✗ 모든 콘텐츠 중앙정렬
✗ 모든 섹션 검은 배경
✗ 모든 카드 icon-title-body 포맷
```

**품질 룰**:
- 각 섹션 *다른* 커뮤니케이션 역할
- 같은 archetype 2회 이상 반복 ✗
- 레이아웃은 *위계 표현*·장식 ✗
- 한 페이지 = 읽기·시각·결정 레이아웃 *mix*
- 밀집 정보 → 표·matrix·다이어그램·사이드 레일
- 감성 콘텐츠 → hero·gallery·poster·editorial
- 제품 콘텐츠 → stage·sticky narrative·visual stack·comparison
